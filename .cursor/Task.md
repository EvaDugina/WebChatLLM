1. Задача
Реализовать небольшое одностраничное веб-приложение — тематический чат с языковой
моделью.

2. Стек
• Бэкенд: Python, любой фреймворк на вкус — Flask, FastAPI, Litestar и т.д.
• Фронтенд: HTML + CSS + JS (без фреймворков)
• LLM: Google Gemini API — есть бесплатная квота, карта не нужна (ai.google.dev). По желанию
можно использовать любой другой сервис и любую другую модель.

3. Функциональность 
- Авторизация по ключу доступа
   Первый экран — форма с единственным полем для ввода ключа доступа.
   Требования к полю:
   • Длина: от 8 до 255 символов
   • Допустимые символы: латинские буквы, цифры, дефис, подчёркивание
   • Кнопка входа заблокирована, если поле не прошло валидацию
   • При вводе некорректного ключа отображается сообщение об ошибке
   Ключ доступа задаётся на стороне сервера (например, через переменную окружения). После
   успешной проверки открывается интерфейс чата.
- Тематический чат с моделью
   Чат работает в рамках определённой темы, заданной системным промптом. Тему выбираете сами.
   Примеры для вдохновения: кулинарный ассистент, обзорщик кода, генератор анекдотов или любая
   другая ваша идея.
   Требования к полю ввода сообщения:
   • Длина: от 1 до 500 символов
   • Пустое сообщение или состоящее только из пробелов — не отправляется
   • Кнопка отправки заблокирована во время ожидания ответа от модели
   • При ошибке запроса к API отображается сообщение об ошибке
   • Контекст модели реализовывать не обязательно
- История переписки
   • Чат один, его сообщения отображаются на странице в формате диалога
   • Каждая запись содержит: дату и время, текст сообщения
   • Сохраняется при перезагрузке страницы и перезапуске проекта (постоянное хранилище —
   файл, база данных или другое решение на ваше усмотрение)

4. Что оценивается
   • Архитектура — структура проекта, разделение ответственности, читаемость кода
   • UX/UI — внешний вид, подача, удобство использования, интуитивность интерфейса
   • Подход к авторизации — как ключ доступа передаётся и обрабатывается между фронтом и
   бэком, как он хранится
